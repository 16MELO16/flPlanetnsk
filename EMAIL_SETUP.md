# Настройка email-подтверждений клиентам

После этой настройки клиент, который укажет email при оформлении заказа, автоматически получит на почту подтверждение со списком позиций, итогом, контактами магазина.

Используем сервис **EmailJS** — у него есть бесплатный тариф 200 писем в месяц, чего обычно хватает небольшому магазину. Если перерастёт — переключимся на платный (~$15/мес за 1000 писем).

Настройка занимает ~15 минут.

---

## 1. Зарегистрироваться на EmailJS

1. Откройте [emailjs.com](https://www.emailjs.com).
2. **Sign Up** → выберите Free Plan.
3. Регистрация по email или через GitHub.
4. После входа окажетесь в дашборде.

## 2. Подключить почтовый сервис, через который EmailJS будет отправлять письма

EmailJS сам не отправляет письма — он использует ваш существующий email-аккаунт (Gmail, Yandex, Mail.ru, etc.). Письма уходят как будто от вас лично.

1. Слева в меню → **Email Services** → **Add New Service**.
2. Выберите вашего почтового провайдера. Самые популярные:
   - **Gmail** — если у вас Google-почта.
   - **Yandex** — если @yandex.ru / @ya.ru.
   - **Mail.ru** — если @mail.ru / @inbox.ru / @list.ru.
   - **Outlook** — если @outlook.com / @hotmail.com.
   - **Custom SMTP** — если корпоративная почта или Rambler/Yandex 360.
3. Авторизуйтесь в выбранном провайдере (откроется его окно).
4. Дайте EmailJS разрешение на отправку писем от вашего имени.
5. Service создан → запомните **Service ID** (что-то типа `service_abc123`).

⚠️ Важно: тот email, который вы подключили — будет адресом отправителя. Клиент увидит письмо «От: Цветочная планета `<flower-planet-nsk@rambler.ru>`». Лучше использовать ваш рабочий магазинный email, а не личный.

## 3. Создать шаблон письма

1. Слева → **Email Templates** → **Create New Template**.
2. Заполните:
   - **Template Name** — `Подтверждение заказа` (любое название).
   - **From Name** — `Цветочная планета`.
   - **To Email** — `{{to_email}}` (двойные фигурные скобки! это переменная).
   - **From Email** — оставьте по умолчанию (ваш магазинный email).
   - **Subject** — `Заказ принят — Цветочная планета`.
   - **Reply To** — оставьте пустым или ваш магазинный email.

3. **Content (тело письма)** — вставьте этот шаблон:

   ```
   Здравствуйте, {{to_name}}!

   Спасибо за заказ в «Цветочной планете». Мы скоро свяжемся с вами для подтверждения.

   Ваш заказ:
   {{order_items}}

   Итого: {{order_total}}

   Комментарий: {{order_comment}}

   Контактный телефон: {{customer_phone}}

   ─────────────────────────────────────
   {{shop_name}}
   📍 {{shop_address}}
   ☎️  {{shop_phone}}

   Если что-то не так — напишите нам на этот email или позвоните.
   ```

   Все `{{переменные}}` будут автоматически заменены на данные конкретного заказа.

4. Если хотите красивое HTML-письмо с логотипом и цветами — переключите редактор в **HTML mode** (вверху справа в редакторе) и используйте HTML-разметку. Но для начала и текстового достаточно.

5. Нажмите **Save**.

6. Запомните **Template ID** (что-то типа `template_xyz789`).

## 4. Получить Public Key

Это ключ, который безопасно публиковать в коде сайта (он только разрешает отправку через ваш сервис, но не даёт доступа к управлению).

1. Слева в меню → **Account** → **General**.
2. Найдите блок **Public Key** — длинная строка вида `aBcDeFgH123IjKlM`.
3. Скопируйте его.

## 5. Вписать три ключа в `index.html`

Откройте `index.html` в редакторе. В начале `<script>` найдите блок:

```js
const EMAILJS_CONFIG = {
  serviceId:  'YOUR_EMAILJS_SERVICE_ID',
  templateId: 'YOUR_EMAILJS_TEMPLATE_ID',
  publicKey:  'YOUR_EMAILJS_PUBLIC_KEY',
};
```

Замените три плейсхолдера на ваши значения из шагов 2, 3, 4. Должно получиться примерно так:

```js
const EMAILJS_CONFIG = {
  serviceId:  'service_abc123',
  templateId: 'template_xyz789',
  publicKey:  'aBcDeFgH123IjKlM',
};
```

Сохраните файл.

Запустите сервер (если ещё не запущен):


Откройте `localhost:8080` → положите цветок в корзину → оформите заказ:

Через несколько секунд на ваш email должно прилететь письмо с темой «Заказ принят — Цветочная планета». Проверьте папку **Спам** — на первом письме иногда туда попадает.

## Что делать, если письмо не пришло

**1. Проверьте папку Спам.** EmailJS-письма с непрогретого аккаунта часто туда попадают. Через несколько отправок репутация выровняется и они начнут идти в Inbox.

**2. Откройте DevTools на сайте → Console.**

При оформлении заказа должна появиться одна из строк:
- ✅ `[email] order confirmation sent to test@example.com` — всё хорошо, письмо ушло.
- ⚠️ `[email] failed to send confirmation: ...` — есть ошибка, прислоните мне.

Если ничего нет — значит EmailJS не настроен (плейсхолдеры YOUR_EMAILJS_* не заменены) или вы забыли сохранить шаблон.

**3. Проверьте лимит.** EmailJS → дашборд → видите счётчик отправленных. Бесплатный лимит 200/мес. Если превысили — письма не уходят.

## Что в письме нельзя:

- **Изменять {{переменные}} в шаблоне без правки кода.** Если переименуете `{{to_email}}` в что-то другое — отправка сломается. Можно добавлять новые переменные в коде (`emailjs.send(serviceId, templateId, { новая_переменная: значение })`) и использовать в шаблоне.
- **Использовать чужой Public Key.** Только ваш собственный — иначе EmailJS не позволит отправлять.

## Что добавлять и менять можно свободно

- **Текст шаблона** — можете писать что угодно, главное оставлять `{{переменные}}`.
- **HTML-разметку** — добавьте логотип, цвета, фоновое изображение.
- **Вторая отправка админу** — можно сделать второй шаблон и в коде вызывать `emailjs.send(...)` дважды: одно письмо клиенту, другое вам, чтобы не зависеть от web3forms.

Если захотите изменить шаблон письма (например, добавить логотип) — скажите, помогу с HTML.
