# Подключение российского CDN вместо Cloudflare

После отключения Cloudflare сайт работает напрямую через GitHub Pages — это решает проблему доступа из России, но мы теряем DDoS-защиту, WAF и кэширование. Эта инструкция описывает, как вернуть эти функции с помощью российских провайдеров.

Два варианта на выбор. Можно реализовать любой — оба работают со стороны России и совместимы с GitHub Pages.

## Вариант A — DDoS Guard (рекомендую для начала)

**Что это:** Российский аналог Cloudflare. Бесплатный тариф включает DDoS-защиту, базовый WAF, минимальное кэширование. Узлы по всей России.

**Цена:** 0 ₽ (бесплатный тариф) или от 500 ₽/мес (Professional с расширенным кэшированием и аналитикой).

**Сложность:** Низкая — настройка похожа на Cloudflare.

### Шаги настройки DDoS Guard

1. Зарегистрироваться на [ddos-guard.net](https://ddos-guard.net) (на русском, в РФ).
2. Подтвердить email.
3. В кабинете — **«Защита сайтов»** → **«Добавить сайт»**.
4. Ввести домен: `flower-planet-nsk.ru`.
5. DDoS Guard выдаст:
   - **IP-адрес для A-записи** (типа `185.178.208.X`) — на него мы направим трафик.
   - **Способ верификации** — либо TXT-запись в DNS, либо файл в корне сайта.

6. Возвращаемся в Cloudflare (он остаётся только как DNS-провайдер):
   - **DNS → Records** → удаляем 4 существующие A-записи (которые указывают на GitHub Pages).
   - **Add record** → A → Name: `@` → IPv4: IP от DDoS Guard → **Proxy: OFF** (серое облачко!) → Save.
   - CNAME `www` оставляем как есть.

7. Через 5-30 минут DDoS Guard увидит, что A-запись указывает на него, и активирует защиту.

8. **На стороне DDoS Guard в кабинете** настраиваем origin (откуда брать настоящий контент):
   - Origin: `16melo16.github.io`
   - SSL: автоматический

После этого: посетители → DDoS Guard → GitHub Pages. DDoS Guard фильтрует атаки, кэширует статику в России.

### Если хочется поменьше телодвижений — оставаться на Cloudflare DNS

Можно оставить домен на nameservers Cloudflare (как сейчас) и просто менять A-запись с IP GitHub Pages на IP DDoS Guard. Cloudflare продолжит работать как DNS-провайдер, но трафик уже не пойдёт через него (proxy off везде).

## Вариант B — BunnyCDN

**Что это:** Глобальный CDN с быстрыми узлами в Москве и Питере. Стабильно работает в России. Не имеет DDoS-защиты на бесплатном тарифе, но трафик кэшируется и сайт быстрее.

**Цена:** $1-3/мес (~80-250 ₽) в зависимости от трафика — это «pay as you go» с минимальным депозитом.

**Сложность:** Средняя — потребуется создать pull-zone и настроить custom hostname.

### Шаги настройки BunnyCDN

1. Зарегистрироваться на [bunny.net](https://bunny.net).
2. Внести депозит $5 на счёт (это минимум, хватит на несколько месяцев).
3. **CDN → Add Pull Zone**:
   - Name: `flower-planet-cdn` (любое)
   - Origin URL: `https://16melo16.github.io`
   - Pricing Tier: Standard
   - Geo-Replication: только Russia + Europe (дешевле)
4. После создания получите CDN-URL типа `flower-planet-cdn.b-cdn.net`. Проверьте — открыв его, должна загрузиться ваша страница.

5. **Hostnames** → **Add Hostname**:
   - Hostname: `flower-planet-nsk.ru`
   - **Force SSL: On**
   - **Add Free SSL Certificate** (Let's Encrypt) — Bunny выпустит сертификат бесплатно.

6. В Cloudflare (или у регистратора) меняем A-записи `flower-planet-nsk.ru` на IP-адреса BunnyCDN. BunnyCDN покажет какие именно IP подставлять — это нужно прочитать в их инструкции при создании Hostname.

7. После обновления DNS (5-30 минут) сайт работает через Bunny.

### Дополнительно в Bunny

- **Edge Rules** → можно настроить кэширование (TTL 1 час для HTML, 1 неделю для статики).
- **Bunny Shield** ($10/мес отдельно) — добавляет WAF и DDoS-защиту, если нужно.

## Сравнение

| Параметр | DDoS Guard Free | BunnyCDN |
|---|---|---|
| DDoS-защита | ✅ Есть | ❌ Только в платной Shield |
| Скорость в РФ | Хорошая | Отличная |
| Цена | 0 ₽ | ~150 ₽/мес |
| Аналитика | Базовая | Подробная |
| Настройка | Простая | Средняя |
| Поддержка | Только на русском | Только на английском |

**Моя рекомендация — начать с DDoS Guard.** Бесплатно, в России, всё на русском, базовая защита есть. Если потом захочется большей скорости — переключиться на BunnyCDN можно за час.

## Что делать перед переключением

1. **Не отключайте Cloudflare полностью** — оставьте на нём DNS, просто proxy выключен (как сейчас). Так удобнее быстро откатиться, если что-то пойдёт не так.
2. **Сначала тестируйте на одной A-записи**. После добавления записи DDoS Guard / BunnyCDN — откройте сайт с нескольких устройств, убедитесь что грузится.
3. **Проверяйте на мобильном из России** — главная цель миграции.

## Что делать если что-то сломалось

Откатиться элементарно: в Cloudflare DNS меняем A-записи обратно на IP GitHub Pages:
- 185.199.108.153
- 185.199.109.153
- 185.199.110.153
- 185.199.111.153

Через 5-15 минут всё работает как до переключения.
